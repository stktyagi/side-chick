The problem

Coding agents have a blind spot

Agents explore the same code repeatedly, burning tokens and losing context. Aide separates exploration from solving.

💥

Without Aide

The main agent wastes its context window on broad file reads and searches, polluting later reasoning.

  • Exploratory reads stay in solver history forever
  • Irrelevant snippets pollute reasoning
  • Same search repeated across tasks
  • Context window fills up fast
⚡

With Aide

Delegate repository exploration to a lightweight subagent that returns only what matters — compact file-line citations.

  • Exploration outside the solver trajectory
  • Compact <final_answer> evidence only
  • Parallel tool calls in a single turn
  • Up to 60.3% fewer main-agent tokens
How it works

Three steps to focused evidence

The main agent delegates a query, Aide explores autonomously, and returns compact citations.

1

Delegate

The main coding agent sends a natural-language context query to Aide — no need to search itself.

2

Explore

Aide uses read-only tools — Read, Glob, Grep — issuing independent calls in parallel across turns.

3

Cite

Returns a compact <final_answer> block with file paths and line ranges. No code dump, no noise.

Features

Built for agent-scale code exploration

🔍

Parallel Tool Calling

Independent Read, Glob, and Grep calls issued in the same turn — faster exploration, fewer round trips.

📎

Compact Evidence

Returns only file paths and line ranges in a machine-readable <final_answer> block. No code snippets.

🧠

Trainable Models

Train 4B–30B exploration models with SFT and task-grounded RL. Pre-trained weights available on Hugging Face.

🌍

Model-Agnostic

Works with any OpenAI-compatible endpoint — OpenAI, Azure, local vLLM, or any custom backend.

🔗

MCP Native

Ships as both a CLI tool and an MCP server for seamless integration with opencode, Claude Code, and more.

🔒

Read-Only by Design

Aide never modifies files. It explores — the main agent edits. Safe for any codebase.

MCP Integration

Plug into any coding agent

Aide runs as an MCP server, giving any MCP-compatible agent supercharged code search.

Seamless agent integration

Add Aide as an MCP tool to opencode, Claude Code, Codex CLI, or any agent that supports the Model Context Protocol.

  • Register as a task tool — no code changes needed
  • Works over stdio or SSE transport
  • Configure endpoint, model, and max turns per task
  • Returns structured citations the main agent can act on
# Start the MCP server
$ aide mcp

# In your agent config (opencode.json):
{
  "mcpServers": {
    "aide": {
      "command": "aide",
      "args": ["mcp"]
    }
  }
}

# Or use directly:
$ aide \
    --query "Find auth logic" \
    --max-turns 6 \
    --citation
